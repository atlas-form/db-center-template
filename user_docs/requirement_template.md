# 需求填写模板

如果你不知道该怎么向 AI 描述需求，可以直接按照这个模板填写。

不需要一次写得很专业，只要把业务说清楚就行。

---

## 模板

```text
我要做一个：<系统 / 功能名称>

业务目标：
<一句话说明这个功能是干什么的>

核心对象：
1. <对象 1>
2. <对象 2>
3. <对象 3>

用户可以做的事情：
1. <动作 1>
2. <动作 2>
3. <动作 3>

是否需要登录：
- 是 / 否

如果需要登录：
- 哪些功能只有登录后才能访问
- 数据是否只能看自己的
- 是否有管理员

数据隔离规则：
- 普通用户是否只能看自己创建的数据
- 管理员是否能看全部数据

你希望系统提供的接口：
1. <接口用途 1>
2. <接口用途 2>
3. <接口用途 3>

补充说明：
<任何你觉得重要的规则>
```

---

## 示例

```text
我要做一个：任务管理功能

业务目标：
让用户可以创建和查看任务

核心对象：
1. 任务

用户可以做的事情：
1. 创建任务
2. 查看自己的任务
3. 修改自己的任务状态

是否需要登录：
- 是

如果需要登录：
- 所有接口都需要登录
- 普通用户只能看自己的任务
- 管理员可以看全部任务

数据隔离规则：
- 普通用户只能访问自己创建的任务
- 管理员可以访问全部任务

你希望系统提供的接口：
1. 创建任务
2. 查询我的任务
3. 修改任务状态
4. 管理员查询全部任务

补充说明：
任务需要有标题、内容、状态、创建时间
```

---

## 把这份模板交给 AI 的方式

填完以后，可以直接这样发给 AI：

```text
这是我的业务需求。

请先不要直接写代码。
先阅读 AI_PROTOCOLS/AI_WORKFLOW.md 和 AI_PROTOCOLS/TABLE_ADDING_PROTOCOL.md。
如果需求涉及登录用户，请继续阅读 AI_PROTOCOLS/AUTH_INTEGRATION_GUIDE.md。

然后根据我的需求，先输出一份“服务端开发文档”，等我确认后再开始开发。

<把你填写好的需求贴在这里>
```
